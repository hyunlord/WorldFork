## 0. 한눈 요약

**작품(이름·입력·업로드)을 받아 → 공개 데이터 + 사용자 입력으로 세계를 세우고 → grounding된 AI GM 서사 게임으로, 어울리는 비주얼·스타일까지 매칭해 플레이하는 엔진.** 웹·모바일. "뤼튼 상위호환." 단 — **심장(서사 엔진)을 한 작품(게임 속 바바리안)에서 재미로 증명한 뒤** 자동화 껍데기를 얹는다. 기술만 쌓고 재미 검증을 미루지 않는다.

---

## 1. 제품 비전

- 한 줄: **누구나 좋아하는 작품/세계를, 살아있는 동료와 함께 자유롭게 플레이하는 AI 서사 게임 엔진.**
- 입력원: ① 작품명(공개 웹 데이터 검색) ② 사용자 입력(원하는 설정·요청) ③ 사용자 업로드 파일(원작 텍스트 등).
- 출력: 원작에 **일관되게 grounding**된 서사 게임 — 자유 입력이 다 반영되고, 히스토리를 기억하고, 어울리는 비주얼·게임 스타일이 매칭됨.
- 비교: 뤼튼류 캐릭터·세계 플레이의 상위호환(게임 시스템·비주얼·메모리·grounding 결합).

---

## 2. 법적 포지셔닝 (★ 설계 제약)

- 생성은 **공개 웹 데이터 + 사용자 입력 + 사용자 업로드 파일**에 한한다.
- **유저 주도**: 플랫폼이 저작권 본문을 직접 소싱·재배포·호스팅하지 않는다. 사용자가 가져오고 지시한다.
- 업로드/원작 본문은 **로컬·사용자 스코프**로 격리(공개·재배포 금지). 현 WorldFork의 `.local`(git 금지) 규율을 엔진 전체 원칙으로 승격.
- 게임 화면 명칭은 사용자 맥락에 따르되, 코드·git·로그는 변환/익명 처리(현 IP 마스킹 원칙 유지).

---

## 3. ★ 핵심 아키텍처 — 범용 엔진 + 콘텐츠팩

### 3.1 범용 엔진 (IP-agnostic, 한 번 만들어 재사용)

- **AI GM**: 장면·서술·NPC·결과 생성. 구조화 출력. 캐논·상태에 고정(freeform 금지).
- **성향 동료**: 5축 자율·지시·반응(순응/변형/거부) — 차별점.
- **기억/히스토리**: 영구 세계 상태 + 사건 로그 + 컨텍스트 관리(요약).
- **RAG grounding**: 소스(업로드 본문/공개 웹)를 인덱싱·검색해 매 장면 반영.
- **자유 입력 해석**: 자연어 → 구조화 행동, 세계 상태에 실제 반영.
- **전투/메커니즘 primitive**: HP·명중·피해·상태이상·드롭(서술/일러스트로 묘사).
- **구조화 규율**: 상태·전환·선택지는 코드 권위, 서술은 옷(챗봇 함정 회피 — NARRATIVE_DESIGN §8).

### 3.2 콘텐츠팩 (작품별 데이터, 갈아끼움)

- **월드 바이블**: 세계·인물·시스템·경제(현 WORLD_BIBLE 형식).
- **캐논 인덱스**: 사건·인물·타임라인 구조화(GM 주입용).
- **소스 grounding**: 업로드 본문 RAG 인덱스(로컬) 또는 공개 웹 파생 요약 인덱스.
- **비주얼·스타일 설정**: 아트 스타일·게임 스타일·UI 테마.

### 3.3 첫 콘텐츠팩 = WorldFork

『게임 속 바바리안으로 살아남기』 = **엔진 증명 무대**. 엔진이 한 작품에서 grounding·자유·기억·재미를 입증하면, 이후 작품은 콘텐츠팩만 교체.

---

## 4. 엔진 재설계 — 3축 (★ 이번 근본 재설계의 본체)

현 build는 "안 멈추게" 하려고 비트를 딱딱하게 박아 **레일 게임**이 됐다. 목표는 자유 + grounding + 기억의 동시 충족.

### 4.1 Grounding (이상한 거 안 나오게)
- **캐논 백본**(월드 바이블 + 구조화 인덱스) + **소스 RAG**(업로드 본문 로컬 인덱스 / 공개 웹 요약 인덱스).
- GM이 매 장면에서 실제 소스를 검색·참조해 반영 → 날조(예: 근거 없는 부족명) 최소화.
- 소스가 약하면(요약만) grounding도 그만큼 얕음 — 정직하게 노출.

### 4.2 자유 + Coherence (입력이 다 반영, 레일 탈피)
- 자유 입력/선택이 **실제 세계 상태를 바꾸고** GM이 반영. 딱딱한 비트 레일 폐기.
- 원작 사건은 강제 레일이 아니라 **끌개**(균형 캐논 — NARRATIVE_DESIGN §4).
- 자유를 키우는 만큼 **상태 + 기억 + grounding이 coherence 가드레일**(멈춤·산만·모순 방지).

### 4.3 기억 (히스토리 다 기억·반영)
- 영구 세계 상태(flags·관계·소지·생사) + **사건 로그** + 길어지면 **요약**(장기 컨텍스트 관리).
- GM 프롬프트에 누적 히스토리 주입 → 이전 일을 기억하고 반영.
- 프로젝트의 RAG·장기 컨텍스트 연구 자료 활용.

## 5. 플랫폼 아키텍처 (웹 + 모바일)

- **백엔드 = 엔진 서비스**(상태·GM·성향·RAG·기억). 클라이언트와 분리.
- **씬 클라이언트**: 웹(지금) → 모바일(이후). 동일 API.
- 표현은 텍스트 + 일러스트 + (선택적) canvas/WebGL. 모바일까지 가는 구조 = 서버 권위 + 얇은 UI(현 v3/gm 분리, 픽셀 RTwP는 레거시 보존).

---

## 6. 비주얼·스타일 매칭

- 콘텐츠팩의 **비주얼·게임 스타일 설정**에 따라 아트(ComfyUI/Flux)·UI·연출 매칭.
- 작품 톤 → 아트 스타일/팔레트/게임 스타일 매핑. **자동 매칭은 후속 단계**(Phase C); 지금은 WorldFork에 수동 설정.

---

## 7. 단계별 로드맵 + ★ 규율

- **Phase A (지금)**: 범용 엔진(§3.1) + WorldFork 팩. **4.1 grounding + 4.2 자유 + 4.3 기억** 재설계. → 한 작품에서 **재미 증명**.
- **Phase B**: 콘텐츠팩 저작 도구 — 업로드/웹에서 월드 바이블·캐논 인덱스 **반자동 생성**.
- **Phase C**: 자동 생성 — 작품명 → 공개 데이터 수집 → 콘텐츠팩 생성 → **스타일·비주얼 자동 매칭**.
- **Phase D**: 모바일 클라이언트.

> ★ **규율**: 껍데기(자동화·범용화)는 **심장(엔진)이 재미로 증명된 다음**에 얹는다. 더 큰 스케일에서 "기술만 쌓고 재미 검증 안 함"을 반복하지 않는다. 정공법·정직 보고·YAGNI·Ship Gate 유지.

---

## 8. 정직한 리스크

- **IP/법적**: 유저 주도·공개 데이터로 완화하나 무결은 아님. 공개 제품화 시 재검토 필요.
- **자동 생성 품질**: 세계관·스타일·비주얼 자동 매칭이 "완벽"하긴 어렵다 — 각각 큰 과제. 점증 검증.
- **자유 ↔ coherence 긴장**: 자유를 키울수록 산만·모순·멈춤 리스크. grounding·상태·기억이 받쳐야 함(이 장르의 본질 난제).
- **비용/지연**: GM·성향·RAG LLM 호출의 지연·비용. 스트리밍·경량 라우팅·캐싱으로 관리.
- **스코프**: 전체는 스타트업 규모 다년 제품. 단계·규율로 분할.
