> **세계관 속에서 사는 자율 파티 시뮬레이션 게임**
> 게임 코어는 코드(즉각·결정적), LLM은 성향 기반 동료의 영혼.
>
> 작성: 2026-06-11 · 상태: V3 — 방향 전환(뒤집기)
> 이전: ROADMAP.md(V1), ROADMAP_V2.md — 턴제 텍스트 어드벤처(폐기)

## 0. 왜 V3 — 무엇이 변했나

### 깨달음 (도그푸딩으로 발견)

V1/V2는 **LLM이 게임을 굴리는** 구조였다(매 행동 LLM 서사 → 선택지). 직접 플레이하니:
- "선택 → 대기 → 선택"의 리듬이 끊겨 **지루**
- 선택지가 상황을 못 따라감(성년식 무기 선택이 전투 중에도 고정)
- LLM 서사는 좋으나, 매번 ~6s 대기가 게임을 죽임

**핵심 실수**: LLM으로 게임을 만들려 했다. 정공법은 — **게임을 게임답게 만들고 LLM을 얹는** 것.

### V3 방향 (뒤집기)

```
게임 코어 = 코드 (즉각·손맛·결정적)
LLM       = 성향 기반 동료의 영혼 (자율 행동 + 지시 해석)
형식       = 실시간 일시정지(RTwP) — 발더스 게이트식, 틱 기반으로 시작
```

비전: **파티원이 성향대로 자율 행동**하고, 플레이어가 **자연어로 개입**하면 그 지시를 **성향대로 해석**(순응/변형/거부). 자유 행동·결과는 **영구**로 세계에 남는다.

상세 설계: `DESIGN_disposition_engine.md`

## 1. 핵심 원칙 (변하지 않을 것)

1. **게임 상태 = 코드** — HP/명중/피해/위치/플래그는 결정적 코드(기존 #5 유지·강화)
2. **LLM = 영혼, 양념** — 매 행동이 아니라 분기점·개입·갈등·결정적 서사만
3. **성향이 좌우** — 자율 행동 + 지시 해석을 성향 모델이 결정
4. **영구성** — 중요한 자유 행동·결과는 세계가 기억(선택적 영구)
5. **틱 기반 시작, 실시간은 나중** — 개인 규모 현실성. 언젠가 실시간 전환
6. **도그푸딩 우선** — 각 단계마다 직접 플레이로 재미 검증

## 2. 폐기 / 보존 / 신규

### 폐기 (V1/V2에서)
- 매 행동 LLM 호출(턴제 서사 루프)
- 정적 선택지 시스템
- "LLM이 게임 진행을 주도" 구조

### 보존 (살림 — 자산)
- 게임 로직 코드(HP/명중/피해/가격 계산)
- GBNF 구조화 출력(자유 행동 → 게임 효과 변환)
- 페르소나 시스템(→ 성향 모델 토대)
- 6축 서사 품질(→ 결정적 순간 서사)
- 던전 2D 맵, 세계관·고증(비요른/1층/방향별)
- 0토큰 분류(→ 코드 기본 패턴의 씨앗)
- 서빙 인프라(Gemma pivotal, 예측/캐시 — 단 LLM 빈도가 줄어 부담↓)

### 신규
- 틱 루프(실시간 일시정지 진행)
- 성향 → 기본 패턴 매핑(코드)
- 지시 해석 엔진(LLM + 가드레일: 순응/변형/거부)
- 파티 시스템(3-4명)
- 영구 상태 모델(world flags / npc memory / relationships)

## 3. 단계별 로드맵 (V3)

> 이전 Tier 0-3(턴제)는 폐기. 새 단계는 **뒤집기 검증 → 비전 정수 → 확장** 순.

### Phase 0 — 성향 코어 (기반)
**목표**: 성향 모델 + 코드 기본 패턴. LLM 없이 동료가 성향대로 움직임.
- 5축 성향 모델(충성/저돌/지혜/변덕/유대)
- 성향 → 기본 행동 패턴 매핑(코드, 즉각)
- 틱 루프 최소(동료 1명 자율 이동/전투)
- **검증**: 동료가 성향대로 자율 행동하는가(코드만으로)

### Phase 1 — 지시 해석 (비전 정수) ★
**목표**: 자연어 개입 → 성향 통과 → 순응/변형/거부.
- 일시정지 + 자연어 입력
- LLM 지시 해석(성향 × 지시 × 상황 → 3갈래)
- GBNF로 행동 변환 → 코드 반영
- 성향 일관성 가드레일(거부가 "성격"으로 읽히게)
- **검증**: 같은 지시에 동료마다 다르게 반응하는가, 납득되는가

### Phase 2 — 영구 세계 (지속성)
**목표**: 자유 행동·결과가 세계에 남음.
- world flags / npc memory / relationships 모델
- LLM이 "무엇이 영구인지" 판정 → 코드 기록
- 재등장 시 코드가 반영(NPC가 기억)
- **검증**: 과거 행동이 나중에 의미 있게 돌아오는가

### Phase 3 — 파티 확장 (규모)
**목표**: 3-4명 파티 + LLM 빈도 튜닝.
- 파티 제어(본인 + 동료 자유 지시)
- 동료 간 상호작용(갈등·협력)
- LLM 호출 빈도 최적화(분기점 감지 규칙)
- **검증**: 파티가 살아있게 느껴지나, 대기가 거슬리지 않나

### Phase 4 — 서사 양념 + 시각 (품질)
**목표**: 결정적 순간 6축 서사 + 시각 에셋.
- 보스/사건/조우 첫 등장 서사(빈도↓, 임팩트↑)
- ComfyUI 시각 에셋 단계적
- **검증**: 빠른 플레이 + 가끔 멋진 서사의 균형

### Phase 5+ — 실시간 전환 (먼 목표)
- 틱 → 진짜 실시간(부드러움)
- 콘텐츠 확장(층/세계관)

## 4. 가장 큰 리스크 (정직)

1. **LLM 호출 빈도** — 파티 × 분기점이 잦으면 비용·대기 폭발. 코드 기본 패턴으로 흡수, 분기점 감지 규칙이 핵심.
2. **예측 불가 ↔ 게임성** — 동료의 지시 거부가 재미 vs 답답함. 성향 일관성으로 납득시켜야.
3. **영구 상태 폭발** — 전부 영구는 관리 불가. 선택적 영구.
4. **실시간 엔진 복잡도** — 틱으로 시작해 위험 회피.
5. **재미 검증** — 매 Phase 도그푸딩. 재미없으면 그 단계에서 멈추고 조정(이번 교훈).
