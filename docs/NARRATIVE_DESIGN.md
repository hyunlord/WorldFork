# WORLDFORK — 서사 게임 설계 (NARRATIVE DESIGN)

> **코어 설계의 단일 진실(Single Source of Truth).** 2026-06-13 확정.
>
> 문서 관계:
> - 본 문서가 **게임 코어**(루프·플레이어 행위·전투·표현)를 정의한다.

## 0. 한눈 요약

AI GM이 원작 『게임 속 바바리안으로 살아남기』 세계를 무대로 장면을 펼치고, 플레이어는 **혼합 입력**(선택지+자유)으로 자유롭게 행동하며, **성향 동료**가 반응하고 세계가 기억한다. 원작의 인물·사건이 **"끌개"**로 등장하되(반가움), 선택으로 운명이 갈린다("뒤바뀐 운명"). 캐릭터는 퍼머데스, 세계·관계·지식은 영속. 전투는 **메커니즘은 코드로, 묘사는 텍스트+일러스트로.**

---

## 1. 정체성

- 한 줄: **AI GM이 끌어가는, 성향 동료와 함께하는 한국어 서사 RPG.**
- 우리만 할 수 있는 이유: 살아있는 **성향 동료**(자율·불신·배신) + **원작 캐논 앵커** + **동적 LLM GM**.
- RTwP 자율 전투에서 전환한 이유: 플레이어의 결정권 부재 = 관전. 비요른마저 AI가 굴려 "한 틱씩 움직이는 화면보호기"가 됐다. 서사 중심은 매 비트가 플레이어의 선택이며, 이 프로젝트의 **원래 정체성(AI GM 서사 엔진)으로의 복귀**다.

## 2. 확정 핵심 결정 (기둥)

| 기둥 | 결정 |
|---|---|
| 코어 패러다임 | **서사 중심** (전투는 보조) |
| 캐논 균형 | **균형** — 원작 사건이 일어나되 선택으로 결과가 크게 갈림 |
| 입력 | **혼합** — 선택지 2~4개 + 자유 입력칸 |
| 동료 | **성향 5축 자율**(불신·배신·영구) — 차별점 |
| 장르 | **하드코어 하이브리드** — 캐릭터 퍼머데스 + 세계·관계 영속 (GAME_DESIGN §6) |
| 전투 묘사 | **텍스트 서술 + Flux/LoRA 일러스트 스틸** (메커니즘은 코드) |
| 표현 | 웹 유력(★ 엔진 최종 보류) / 픽셀 타일 셔플 봉인 |

## 3. 핵심 루프

### 3.1 매크로 (원작 구조를 따름)

성지(튜토리얼) → 미궁 1층(수정 동굴) → 거점(라프도니아·노아르크: 길드·주점·거래·관계) → **미궁 한 달 주기**(개방기 공략 / 폐쇄기 거점 생활) → 캐논 사건 끌개 → 거점·정치·확장. 원작 동심원(성지→미궁→도시→대륙→신세계→궁정)을 단계 해금.

### 3.2 마이크로 (비트 = 한 장면)

1. **GM이 장면 제시** — 서술 + 배경 아트 + 등장인물 포트레이트.
2. **플레이어 혼합 입력** — 선택지 또는 자유 입력.
3. **판정** — 상태·성향·캐논 반영. 전투면 메커니즘 엔진(§6 문서 아님, 본문 6장).
4. **성향 동료 반응** — 순응/변형/거부 + 근거 (기존 엔진).
5. **결과를 세계에 기록** — flags·관계·npc memory (영구).
6. **GM이 다음 비트로** — 캐논 끌개를 향해 부드럽게 견인하되 강제하지 않음.

---

## 4. 균형 캐논 — 끌개(Attractor) 모델

- **문제**: 자유 ↔ 캐논은 상충한다(완전 자유면 캐논이 안 나오고, 완전 캐논이면 레일).
- **해법**: 캐논 사건/인물을 **끌개·세트피스**로 둔다. 원작 구조를 따라 적절한 조건에서 트리거(미궁 개방, 에르웬 밤친구 조우, 마르토앙 남작 사건 등). 발생은 예약되되 **전개·결과는 자유 행동이 결정**. 캐논 이탈(분기)은 버그가 아니라 기능 — 원작 자신의 주제 "뒤바뀐 운명".
- **끌개 3요소**: ① 트리거 조건 ② 부드러운 견인(다른 데로 가도 강제 안 함, 후에 다른 형태로 재등장 가능) ③ 분기 허용.
- **반가움의 원천**: 낯익은 인물·사건이 등장하되, 너의 선택으로 다르게 흐른다.

---

## 5. 성향 동료 — 서사의 심장

- 5축(충성/저돌/지혜/변덕/유대) 자율 (기존 Phase 0-3).
- 너의 선택에 **성향대로 반응**(순응/변형/거부 + 근거).
- 관계 누적(신뢰↔불신), **배신·이탈 가능**, 영구 기록(npc memory).
- 원작 동료(에르웬·아이나르·한스·도라 등)를 성향으로 매핑 (WORLD_BIBLE §11).
- **통제 불완전성 = 드라마.** 원작 정서 "동료를 믿지 마"가 긴장 축.

---

## 6. 전투 — 텍스트 + 일러스트

- ★ **메커니즘 ≠ 묘사 분리.**
- **메커니즘(코드, 속으로)**: 슬라이스 엔진 재사용 — HP/명중/피해/출혈/성향 행동/드롭/정수. 선택의 무게를 보장.
- **흐름**: GM 상황 묘사 → 혼합 입력 전술 결정("도끼로 돌격"/"에르웬에게 엄호"/자유) → 메커니즘 판정(성향 반영) → GM이 결과를 생생히 서술 + 핵심 컷 일러스트.
- **묘사**: 소설식 서술(원작이 가장 잘하던 것) + Flux/비요른 LoRA 일러스트 스틸(배경·전투 CG·표정 포트레이트).
- **정직한 한계**: 일러스트는 생성에 수초 → **핵심 순간의 스틸 컷**이지 실시간 애니메이션이 아님(사전 생성 아트 + 비트 일러스트 + 생생한 텍스트의 조합).
- **픽셀 타일 셔플 봉인.** 보스/세트피스용 픽셀·전술 표현 도입 여부는 열린 결정(11장).

---

## 7. 하드코어 하이브리드 (로그라이크 프레임)

GAME_DESIGN §6 그대로:
- **사망 시 소멸(캐릭터 귀속)**: 빌드·정수 능력·장비·소지금·미궁 진행·직위.
- **영속(플레이어 귀속)**: 관계·npc memory·세계 flags·지식·평판·동료 생사 명부.
- 동료 **영구 사망·배신** 가능.
- **메타 정당화**: 관리자·빙의·다중 플레이어 → 세계는 기억, 캐논은 반복, 캐릭터의 운명만 갈린다. 회차마다 "뒤바뀐 운명".

---

## 8. ★ 챗봇 함정 회피 (게임답게 만드는 장치)

LLM 서사 게임이 흔히 빠지는 함정과 방어책. **이 장이 "재밌는 게임 ↔ 형체 없는 챗봇"을 가른다.**

| 함정 | 방어 |
|---|---|
| 두서없는 GM | WORLD_BIBLE + **구조화 세계 상태**에 GM을 고정(freeform 금지). 캐논 인덱스 참조. |
| 안 중요한 선택 | 매 비트가 **명시 상태**(flags/관계/자원/마석)를 갱신, 결과가 나중에 표면화. 선택은 추적·기록된다. |
| 강제된 캐논 | **끌개 모델**(트리거 + 부드러운 견인 + 분기 허용). |
| "그냥 챗봇" | 서사는 **진짜 시스템**(상태·성향·자원·경제·퍼머데스·진행)의 외피. 게임 시스템이 먼저, 서술은 옷. |
| 페이싱 붕괴 | 거점(이완) ↔ 미궁(긴장) 리듬. GM이 장면 길이·고조 관리. |
| 일관성·지속 실패 | 영구 세계(Phase 2)로 상태·기억 유지. |

## 9. 기술 아키텍처 (현 자산 매핑)

| 레이어 | 역할 | 현 자산 |
|---|---|---|
| **AI GM** | 장면·서술·NPC·결과 생성, WORLD_BIBLE 고정 | ★ 신규 핵심 (LLM) |
| **캐논 앵커** | 원작 인물·사건·설정 인덱스 | WORLD_BIBLE.md (GM용 구조화 필요) |
| **자유 행동 해석** | 자연어 입력 → 구조화 행동 | NL-파싱 연구자료 |
| **성향 동료** | 5축 자율·지시·반응 | Phase 0-3 (완성) |
| **영구 세계** | flags·관계·npc memory | Phase 2 (완성) |
| **전투 메커니즘** | HP/명중/출혈/드롭/성향 | 슬라이스 엔진 (완성) |
| **표현** | 텍스트+일러스트 UI | 웹(유력·보류) + Flux/LoRA 자산 |
| **로그라이크 세이브** | PER-RUN / PERSISTENT | §6 (설계 완료, 구현 예정) |

→ **새로 지을 핵심 = AI GM 오케스트레이션 레이어.** 나머지는 대부분 재사용.

## 10. 표현층 & 엔진

- 서사+텍스트+일러스트엔 **웹이 최적**(브라우저 = 텍스트·이미지·입력 엔진; 비주얼노벨·인터랙티브 픽션의 본령). Godot의 근거(실시간 액션 렌더)는 본 방향에서 소멸.
- ★ **엔진 최종 결정은 보류**(사용자 재고 중). 더 화려한 연출이 필요하면 **브라우저 내 canvas/WebGL(PixiJS 등)**로 가능 — 웹 이탈 불요.
- **픽셀 타일 셔플 봉인**(전투 묘사로 부적합). 기존 픽셀/렌더 자산은 보존, 추후 거점/탐험 맵·세트피스에 재활용 가능.

## 11. 열린 결정 / 다음 단계

**열린 결정**
- 엔진/표현층 최종(웹 유력, 보류).
- 캐논 이벤트 인덱싱·트리거 설계, GM 프롬프트 구조.
- 일러스트 파이프라인(사전 생성 vs 온디맨드), LLM 모델 배치(GM=Gemma 등)·비용.
- 전투의 픽셀/전술 드롭인 여부(보스 한정 등).

**다음 단계 (권장 순서)**
1. **캐논 앵커 구조화** — WORLD_BIBLE을 GM이 쓸 이벤트·인물 인덱스로.
2. **AI GM 수직 슬라이스** — 성지 튜토리얼~미궁 1층 한 시퀀스(장면→혼합 입력→성향 반응→결과 기록, 텍스트+일러스트).
3. 전투 메커니즘을 서술+일러스트 표현에 배선.
4. 거점·경제·미궁 주기.

> ★ 이전처럼 "시스템만 만들고 재미 검증 안 함"을 반복하지 않는다. **GM 슬라이스로 "서사가 재밌나"부터 싸게 검증**하고 확장한다.
