## 1. 현재 상태 (어디 있나)

**방향**: 범용 AI 서사 게임 엔진 + 콘텐츠팩(ENGINE_ARCHITECTURE). WorldFork(『게임 속 바바리안』) = 첫 콘텐츠팩 = 엔진 증명 무대.

**보유 자산 (재사용)**:
- 성향 엔진: 5축 자율 · 지시 해석(순응/변형/거부) · 영구 세계(flags/관계/npc memory) · 파티
- 전투: 메커니즘 primitive(HP/명중/출혈/치명타/드롭) · 내러티브 턴 전투
- AI GM: 구조화 GM(narration + 코드 권위 상태/선택지) · 캐논 앵커 · Gemma 8085 라우팅 · 스트리밍(guided)
- /gm 오프닝 슬라이스: 성인식→미궁→첫 조우→마무리 · 코드 비트 전환 · 혼합 입력 · 카이라 성향 반응
- 인프라: world_memory · IP 마스킹(화면=원작명) · 서사 UI 컴포넌트 · intent/freeform
- /v3 픽셀 RTwP: 레거시 보존(서사엔 봉인)

**현 한계 (= 재설계 이유)**:
- 레일: 비트 스캐폴드가 딱딱해 자유 입력이 흐름을 못 바꿈
- grounding 얕음: GM이 압축 fact만 보고 날조(소설과 어긋남)
- 기억 미활용: 영구 상태는 있으나 히스토리 누적·반영 약함
- (해결됨: 영속 분리 버그 · choice-id 불일치 · raw flag 패널 · 오프닝 캐논 충실)

## 3. ★ Phase A 분해 (근시일 — 지금 작업)

각 단계는 ENGINE_ARCHITECTURE §3~4에 매핑. 슬라이스 단위로 끊어 진행(의뢰서별), 단계마다 정직 보고.

- **A1 — 콘텐츠팩 구조 분리**: 엔진(generic) ↔ WorldFork 팩(data) 분리. (§3) → 이후 작품은 팩만 교체.
- **A2 — 소설 RAG grounding**: 업로드 본문 로컬 인덱스 → 장면별 검색·주입. (§4.1) → 날조↓, 소설 정합↑.
- **A3 — 자유 + de-rail**: 자유 입력/선택이 세계 상태를 실제로 바꾸고 GM이 반영. 캐논=끌개. 상태·기억=가드레일. (§4.2) → "입력이 다 반영."
- **A4 — 기억/히스토리**: 사건 로그 + (길어지면) 요약 → GM 주입. (§4.3) → "이전 일을 기억·반영."
- **A5 — 재미 검증**: 재설계된 오프닝(+ 에르웬 밤친구 비트) 플레이·튜닝 → "서사가 재밌나" 판단.

> 순서·슬라이싱은 의뢰서별로 확정. A1(구조)부터 또는 A2(grounding)부터가 자연스러운 시작점.

## 4. 규율 (유지 — 이전 여정의 교훈)

- **재미 먼저**: 껍데기(자동화)는 심장(엔진)이 재미로 증명된 다음. "기술만 쌓기" 금지.
- 정공법: 플레이스홀더·made-but-never-used 금지, 프로덕션 호출자만, **실패 정직 보고**, YAGNI.
- 검증: mypy --strict + ruff · **Ship Gate 95+**(SKIP 금지) · 실제 플레이 재현으로 확인(유닛 통과 ≠ 작동).
- IP: 화면=원작명 / 코드·git=변환명 / 원작 본문 `.local`(git 금지).
