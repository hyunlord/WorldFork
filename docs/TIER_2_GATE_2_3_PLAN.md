## 1. ★ 사람 검증 정공법 (본인 #16)

### 1.1 사람 검증 ≠ 파이썬 CLI 실행

```
❌ 파이썬 CLI 실행 = "본인 풀 플레이"
   - python tools/play_w2_d5.py → 자동 검증 도구
   - 친구는 파이썬 코드 돌릴 수 없음
   - 본인이 돌려도 "게임처럼" 검증 X (텍스트 출력만)

✅ 사람 검증 = Web UI에서 진짜 게임처럼 플레이
   - 브라우저에서 작품명 입력 → 플랜 검토 → 게임 진행
   - 본인 + 친구 모두 동일 인터페이스
   - 정성 피드백 (재미/UX) 의미 있음
```

### 1.2 실행 시점 — 4 게이트 모두 통과 후만

```
게이트 1   : Layer 2 자동 검증 통합          (★ Tier 1.5 D4 완료)
게이트 1.5 : Layer 1 자동화 (verify/hook/CI) (★ Tier 1.5 D1-D3 완료)
게이트 2   : 게임 완성도 (30턴 + 풍부)         (Tier 2 전반)
게이트 3   : Web UI (브라우저 플레이 가능)      (Tier 2 중반)

→ 게이트 2+3 모두 통과 = ★ 사람 검증 시작 (Tier 2 후반)
```

### 1.3 게이트 통과 전 절대 사람 초대 X

이유:
- "재미없다" 피드백 받아도 게임 미완성 / UI 없음으로 원인 분리 불가
- 친구 토큰 1회성 — 미완성 단계에서 소진하면 회복 불가
- 파이썬 CLI = 본인 도그푸딩만, 친구 검증 X

## 2. 게이트 2 — 게임 완성도

### 2.1 정의

> 30턴 동안 풍부하고 일관된 게임 진행이 가능한 상태.

자동 검증 + 본인 도그푸딩(파이썬 CLI)으로 측정.
사람 검증 X (게이트 3 전까지).

### 2.2 작업 항목 (예상)

```
2.2.1 게임 진행 안정화
  - 30턴 완주율 90%+ (자동 측정, AI Playtester)
  - Mechanical 통과율 80%+
  - 응답 잘림 검출 + 자동 재시도 작동
  - 컨텍스트 폭증 X (KV cache 추적)

2.2.2 응답 풍부도
  - 묘사 톤 일관성 (캐릭터 페르소나 유지)
  - 분위기/장면 전환 자연스러움
  - 짧은 user 액션도 적절한 응답 길이 (verbose 41% → ~10%)

2.2.3 컨텐츠 다양화
  - 다양성 4축 중 최소 1축 본격 작동
    예: 장르 시스템 (모험/관계/추리) 1개 시작
  - 같은 작품 + 다른 장르 = 다른 게임 검증 (자동)

2.2.4 세계관 일관성
  - World Canon 정립 (작품 정보 → canon)
  - LLM 환각 검출 (없는 캐릭터/설정 도입 X)
  - Plan Verify 작동 (Cross-Model)

2.2.5 검증 자동화 (Tier 1.5 D4 자산 활용)
  - GameLoop verify_passed 기반
  - PlaytesterRunner Mechanical 자동 검출
  - TaskContext 매 턴 추적
```

### 2.3 통과 조건

```
✅ AI Playtester 6 페르소나 × 30턴 × 1주 = 90%+ 완주
✅ Mechanical 통과율 80%+ (회귀 측정)
✅ 응답 잘림 < 5%
✅ 1시간 연속 플레이 OOM/메모리 누수 X
✅ Layer 1 ship gate 매 commit 통과
✅ 본인 도그푸딩 (파이썬 CLI)으로 30턴 풀 완주 5회
   ★ 단, 이건 사람 검증 X (자동 검증 보강)
```

### 2.4 자동 검증 vs 사람 검증 분리

```
게이트 2 = 자동 검증만:
  - AI Playtester (Cross-Model 페르소나)
  - 본인 파이썬 CLI 도그푸딩 (자기 점검)
  - Eval Smoke + Eval Set 회귀

게이트 2 ≠ 사람 검증:
  - 친구에게 "재미있나?" 물어봄 ❌
  - 베타 테스트 ❌
  - SNS / Discord 공개 ❌
  - 모두 게이트 3 통과 후
```

## 3. 게이트 3 — Web UI

### 3.1 정의

> 브라우저에서 진짜 게임처럼 플레이 가능한 UI.

본인 + 친구가 동일 인터페이스로 접근 가능.
사람 검증의 전제 조건.

### 3.2 옵션 분석

| 옵션 | 장점 | 단점 | 비용 (작업) |
|---|---|---|---|
| **A. Streamlit** | 가장 빠름, 파이썬 재사용 | 게임 UX 한계 (chat 위주), 모바일 약함 | ~1주 |
| **B. FastAPI + 정적 HTML** | 가벼움, 외부 의존 적음 | UX 디테일 작업 직접 | ~2주 |
| **C. FastAPI + Next.js** | UX 본격, 모바일 OK | 외부 패키지 (next, react), 시간 | ~3-4주 |

### 3.3 선택 기준 (★ 본인 #16 + YAGNI)

```
필수:
  - 브라우저에서 작품명 입력 → 플랜 검토 → 게임 30턴 플레이
  - 점수/mech 시각화
  - 친구가 URL만으로 접근 가능

미필수 (Tier 3+):
  - 회원가입 / 인증
  - Save/Load DB 영속
  - 모바일 최적화
  - 디자인 polish

선택 가이드:
  Tier 2 게이트 3 통과 = "게임처럼 플레이 가능"이지 "출시 품질" X
  → 가장 빠른 옵션부터 (Streamlit or FastAPI+HTML)
  → Tier 3 출시 준비 시 Next.js로 마이그레이션
```

### 3.4 임시 제안 (확정은 Tier 2 진입 시)

```
Tier 2 중반: FastAPI + 정적 HTML (옵션 B)
  - 외부 패키지 1개 추가 (fastapi, jinja2 정도)
  - 게임처럼 동작하는 최소 UI
  - 점수/mech/turn 시각화
  - 친구에게 URL 공유 가능

Tier 3 출시 준비: Next.js로 마이그레이션 (옵션 C)
  - 모바일 + UX polish
  - 회원가입 / 결제 (Tier 3+)
```

### 3.5 통과 조건

```
✅ 브라우저에서 작품명 입력 → 게임 시작 가능
✅ 30턴 동안 끊김 X
✅ 점수/Mechanical 결과 가시화
✅ 본인 + 친구 동일 URL 접근 가능
✅ Layer 2 자동 검증이 Web UI에서도 작동
✅ Playwright E2E 테스트 1개+ 작동
```

## 4. 사람 검증 시작 (게이트 2+3 후)

### 4.1 본인 도그푸딩 (Web UI)

```
조건: 게이트 2+3 모두 통과
  - 본인 5회 풀 플레이 (Web UI, 30턴 완주)
  - 정성 회고 (재미 / 어색함 / 막힘)
  - AI Playtester가 못 잡는 정성 이슈 발견
```

### 4.2 친구 베타 (Web UI)

```
조건: 본인 도그푸딩 통과 후
  - 친구 3-5명 초대
  - 작품명만 입력 → 끝까지 플레이 시도
  - 친구 3명 이상 끝까지 완주 = ★ Tier 0 졸업 조건 충족
  - 정성 피드백 평균 "재미있다" 이상
```

### 4.3 인간 피드백 우선 (★ 메타 14.5)

```
AI 1000회 시뮬 < 인간 1명의 "와닿지 않아요"

게이트 2+3 통과 후:
  - AI = 양/회귀 (자동 검증)
  - 인간 = 질/재미 (사람 검증)
  - 정성 피드백 우선순위 > 정량 점수
```

## 6. ★ 본인 #16 재확인

```
이 문서가 지키려는 정공법:

1. 사람 검증 = Web UI 거의 완성 후만
2. 파이썬 CLI = 자동 검증 / 본인 도그푸딩 도구
3. 친구 토큰 1회성 — 게이트 통과 후 본격 사용
4. 자동 검증과 사람 검증 명확히 분리
5. "게이트 통과 = 다음 단계 진입 자격" (자료 메타 2)

이 정공법 미준수 시:
- "재미없다" 피드백을 미완성 탓으로 돌리기
- 친구 토큰 소진 후 게이트 작업
- 사람 검증 의미 없는 단계에서 강행
- → ★ Tier 0~1.5 모든 작업 회수 불가
```
