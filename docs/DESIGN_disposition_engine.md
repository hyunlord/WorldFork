# 성향 엔진 설계도 — 자율 파티 + 자연어 개입

> WorldFork 뒤집기의 심장. 파티원이 **성향대로 자율 행동**하고, 플레이어가 **자연어로 개입**하면 그 지시를 **성향대로 해석**(순응/변형/거부)한다.
>
> 핵심 원칙: 게임 코어는 코드(즉각·결정적), LLM은 성향 판단(자율 행동 + 지시 해석)의 영혼.
> 작성: 2026-06-11 · 상태: 설계 v1 (뒤집기 청사진)

---

## 0. 한 문장 요약

```
파티원 = 성향 모델(코드) + 성향 판단(LLM)
평소: 성향대로 자율 (대부분 코드 패턴, 가끔 LLM)
개입: 자연어 지시 → 성향 통과 → 순응/변형/거부
결과: 코드가 게임 상태에 반영 (영구)
```

LLM이 못 하던 걸 억지로 시키는 게 아니라, **LLM만 할 수 있는 것**(성향 기반 판단·예측 불가성)을 게임의 심장으로 둔다.

---

## 1. 성향 모델 (각 파티원)

기존 페르소나 자산(`03_character_and_persona.md`)을 게임용으로 압축. WorldSim의 187 traits는 과함 — 게임 판단에 직접 쓰이는 **소수 축**만.

### 제안 축 (5개, 각 0-100)

| 축 | 낮음 ←→ 높음 | 영향 |
|---|---|---|
| **충성** | 독립적 ←→ 헌신적 | 지시 수용도 (낮으면 자기 판단) |
| **저돌성** | 신중 ←→ 저돌 | 자율 전투 행동 (돌격/정찰) |
| **지혜** | 단순 ←→ 영민 | 특이 지시의 위험 간파 (거부 근거) |
| **변덕** | 일관 ←→ 변덕 | 예측 가능성 (높으면 예상 밖) |
| **유대** | 냉담 ←→ 친밀 | 플레이어·동료와의 관계 가중 |

> 5축은 출발점. 게임 테스트로 가감. 각 파티원은 이 축 + 한 줄 배경(예: "전직 도굴꾼, 함정에 예민")을 가진다.

### 성향 → 두 출력

```
성향 ──┬─→ 자율 행동 결정 (평소, 틱마다)
       └─→ 지시 해석 (개입 시, 순응/변형/거부)
```

---

## 2. 틱 루프 — 코드 vs LLM 분담 (★ 비용 핵심)

가장 큰 긴장: 파티 3-4명 × 매 틱 LLM = 호출 폭발(비용·대기). 해결 — **평소 코드 패턴, LLM은 결정적 순간만**.

### 분담 표

| 상황 | 처리 | 이유 |
|---|---|---|
| 이동·추적·기본 공격 | **코드** (성향별 기본 패턴) | 즉각·반복 — LLM 불필요 |
| 적 AI / HP·명중·피해 | **코드** (이미 구현) | 결정적 게임 로직 |
| **새 상황 판단** (분기점) | **LLM** (성향 → 선택) | 예: 함정 발견, 동료 위기, 보물 |
| **플레이어 개입(지시)** | **LLM** (성향 → 해석) | 핵심 — 순응/변형/거부 |
| **갈등/대화** (동료 간, NPC) | **LLM** (성향 → 발화) | 살아있는 대화 |
| 잡몹 반복 전투 | **코드** + 가끔 LLM 서사 | 빈도↓ |

### 성향별 기본 패턴 (코드 — LLM 없이)

```python
# 평소 틱: 성향으로 기본 행동 결정 (LLM 호출 X)
def default_action(party_member, world):
    if world.enemy_near and member.저돌성 > 60:
        return "돌격"
    if world.enemy_near and member.저돌성 < 40:
        return "엄폐 후 원거리"
    if world.unexplored and member.지혜 > 60:
        return "정찰"
    # ... 성향 → 패턴 매핑 (결정적, 즉각)
```

LLM은 **이 패턴으로 안 풀리는 분기점**(새 상황, 개입, 갈등)에서만 호출 → 비용·대기 최소.

### 틱 주기

```
틱 = 짧은 게임 시간 단위 (예: 1틱 = 1초 게임 내)
평소: 코드 패턴으로 N틱 빠르게 진행
이벤트(적 조우/함정/개입): 일시정지 → LLM 판단 → 재개
```

> 1번 결정(영구) + 3번 선택(틱) + 2번(자유 지시)이 여기 녹음.

## 3. 지시 해석 — 순응 / 변형 / 거부 (★ 비전 정수)

플레이어가 일시정지 후 자연어 지시. LLM이 **성향 × 지시 × 상황**으로 반응 결정.

### 흐름

```
[플레이어] "철수, 저 좁은 틈으로 먼저 들어가 정찰해"
        ↓
[LLM 판정] 철수 성향(충성 30, 지혜 80, 변덕 20) × 지시 × 상황(좁은 틈=함정 의심)
        ↓
[반응 결정] 3갈래:
  ├─ 순응   (충성↑ or 안전): 지시대로 정찰
  ├─ 변형   (지혜↑): "먼저 횃불을 던져보고 들어가겠소" (더 나은 방법)
  └─ 거부   (지혜↑ + 위험): "저긴 함정 냄새가 나오. 다른 길을 찾겠소"
        ↓
[게임 반영] 코드가 선택된 행동을 상태에 적용 (영구)
```

### 반응 결정 규칙 (LLM 프롬프트 가드레일)

- **순응**: 충성 높거나, 지시가 성향과 충돌 안 하거나, 위험 낮음
- **변형**: 지혜 높아 더 나은 방법 알거나, 부분 동의
- **거부**: 지혜로 명백한 위험 간파 + 충성 낮음, 또는 성향과 정면 충돌(겁쟁이에게 자살 돌격 등)
- **일관성**: 반응에 **성향 근거를 한 줄로** 보이게("나는 신중하니…") → 플레이어가 납득
- **변덕 축**: 높으면 위 규칙에 예측 불가 요소 가미(가끔 뜻밖)

> 핵심: 거부·변형이 "버그"가 아니라 "성격"으로 읽히게. 일관성이 재미의 열쇠.

## 4. 영구 반영 — 세계가 기억한다 (1번 선택)

자유 행동·지시 결과가 **영구**로 남는다. 단 전부는 관리 폭발 → **선택적 영구**.

| 영구 대상 | 예 | 저장 |
|---|---|---|
| 세계 변화 | 무너뜨린 천장, 태운 다리 | world flags |
| NPC 기억 | "넌 날 배신했지" | npc memory |
| 관계 변화 | 동료 유대 ±, 신뢰 | relationship state |
| 캐릭터 성장 | 트라우마, 새 습관 | persona delta |
| 일회성(영구 X) | 이번 전투 버프 | 휘발 |

```python
# 영구는 코드 상태에. LLM은 "무엇이 영구인지" 판정만.
class WorldState:
    flags: dict          # 세계 변화
    npc_memories: dict   # NPC가 기억하는 사건
    relationships: dict  # 관계
# LLM 판정 → 코드가 여기 기록 → 다음 등장 시 코드가 반영
```

## 5. 전체 데이터 흐름

```
        ┌─────────────────── 평소 (코드, 즉각) ───────────────────┐
        │  틱 → 성향별 기본 패턴 → 이동/전투 → 상태 갱신 → 다음 틱  │
        └──────────────────────────┬───────────────────────────┘
                                    │ (분기점 / 플레이어 개입)
                                    ↓ 일시정지
              ┌──────────── LLM 성향 판단 ────────────┐
              │  자율 분기: 성향 → 선택                 │
              │  개입 지시: 성향 × 지시 → 순응/변형/거부 │
              │  대화/갈등: 성향 → 발화                 │
              └──────────────────┬────────────────────┘
                                 ↓ (구조화 출력 — GBNF)
              ┌──────── 코드 반영 (결정적) ────────┐
              │  행동 → 게임 상태 → 영구 기록(선택적) │
              └──────────────────┬────────────────┘
                                 ↓ 실시간 재개
                              (가끔 LLM 서사 — 결정적 순간만)
```

## 7. 열린 설계 질문 (다음 토론)

1. **성향 축 확정** — 위 5축으로 충분한가? 게임 판단에 더 필요한 축은?
2. **LLM 호출 빈도** — "분기점"을 코드가 어떻게 감지해 LLM을 부를까(규칙)? 너무 잦으면 비용, 너무 드물면 죽은 동료.
3. **변덕의 정도** — 예측 불가가 재미 vs 답답함의 경계는? 테스트로 튜닝.
4. **파티 규모** — 3명 vs 4명. 많을수록 LLM 부하 ×.
5. **개입 빈도** — 플레이어가 얼마나 자주 끼어들 수 있나(쿨다운? 자유?).

## 8. 다음 단계 (구현 순서 제안)

```
1. 성향 모델(코드) — 5축 + 기본 패턴 매핑          [작게]
2. 틱 루프 프로토타입 — 동료 1명 자율 이동/전투     [핵심 검증]
3. 지시 해석 — 자연어 → 순응/변형/거부 (1명)        [비전 정수]
4. 영구 상태 — flags/memory 최소                    [지속성]
5. 파티 확장 — 3명 + LLM 빈도 튜닝                  [규모]
6. 서사 양념 — 결정적 순간 6축 (빈도↓)              [품질]
```

각 단계 후 **직접 플레이로 재미 검증**(도그푸딩). 재미없으면 그 단계에서 조정.
